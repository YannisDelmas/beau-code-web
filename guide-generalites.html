BCW-1 Recommandations générales pour le codage web

Noms de fichiers

Les noms des fichiers et dossiers utilisés sur le web doivent respecter les règles suivantes.

  • Écriture en kebab-case, sans espace ni ponctuation.
  • Il est recommandé de ne pas dépasser 32 caractères, extension comprise.
  • Les mots employés doivent refléter de façon claire et compréhensible le contenu ou la fonction dans le site du fichier ou dossier.
  • Une ou plusieurs extensions indiquent le format d'un document.

Le chemin complet, depuis la racine, ne doit pas dépasser 127 caractères, séparateurs compris.

Il est possible d’utiliser plusieurs extensions pour indiquer un encodage, une langue, etc. Le principal serveur web, Apache, est capable d’adapter sa réponse à l’aide de ces informations [Apache Mod_mime].

Pour des raisons de sécurité, les administrateurs web peuvent activer la directive X-Content-Type-Options: nosniff qui demande au navigateur de respecter strictement les types MIME indiqués par le serveur web. [MDN X-Content-Type-Options] [Opquast Règle 201]
