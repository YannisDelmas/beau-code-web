BCW-2 Recommandations pour le codage en HTML

Validité du code

  1. L'encodage doit être précisé [Opquast Règle 225]. Recommandé : UTF-8, sans BOM [Opquast Règle 226].

    À défaut d'UTF-8, l'encodage devrait être, au moins, en Unicode.

    Outil de vérification : W3C Internationalization Checker.

  2. Le balisage doit être valide. En particulier, les éléments doivent être correctement imbriqués et les caractères spéciaux doivent toujours être correctement échappés par des entités, en particulier les & dans les adresse.

    Les caractères invisibles (p. ex. &nbsp;) devraient également l'être, mais pas les autres caractères (p. ex. € ou ⓒ).

    Outils de vérification : affichage de code des principaux navigateurs, vérificateur syntaxique de l'éditeur de code, W3C Markup Validation Service.

  3. Le code devrait être conforme à une version choisie du format choisi. Format recommandé: HTML5 en syntaxe HTML.

    En particulier, un identifant HTML (id) n'est utilisé qu'une fois au plus par page [Opquast Règle 229].

    Outil de vérification : W3C Markup Validation Service.

  4. Les éléments devraient toujours avoir une balise de fermeture, sauf éventuellement pour les éléments toujours vides (link, img, br).

  5. Les valeurs d'attribut doivent toujours être entre guillemets (simple ' ou double "), sauf peut-être pour les attributs booléens (p. ex. required, disabled ou open).

    Si le code à être intégré à des pages XHTML (p.ex. un composant réutilisable), il est recommandé d'utiliser une écriture compatible avec le XML, notamment :

    • Noms de balises et d'attributs en minuscules (le XML est sensibles à la casse).
    • Les attributs, même booléens, ont toujours une valeur, vide si besoin.
    • Les commentaires ne contiennent pas la séquence --.

    La syntaxe XHTML est déconseillée aux concepteurs débutants.
