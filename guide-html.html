BCW-2 Recommandations pour le codage en HTML

Validité

  1. L'encodage DOIT être précisé. RECOMMANDÉ: utf-8 (sans BOM).

  2. Le balisage DOIT être valide. En particulier, les éléments DOIVENT être correctement imbriqués et les caractères spéciaux DOIVENT toujours être correctement échappés par des entités, en particulier les & dans les URI.

    Les caractères invisibles (p. ex. &nbsp;) DEVRAIENT également l'être, mais pas les autres caractères (p. ex. € ou ⓒ).

    Outils: navigateur, vérificateur de l'éditeur de code.

  3. Le code DEVRAIT être conforme à une version choisie du format choisi.Format RECOMMANDÉ: HTML5 en syntaxe HTML.

    Outils: valideur W3C

  4. Les éléments DEVRAIENT toujours avoir une balise de fermeture, sauf éventuellement pour les éléments toujours vides (link, img, br).
  5. Les valeurs d'attribut DOIVENT toujours être entre guillemets (simple ou double), sauf peut-être pour les attributs booléens (p.ex. required, disabled, open).

    Si le code a vocation à être lu informatiquement ou à être intégré à des pages XHTML (p.ex. un composant réutilisable), il est RECOMMANDÉ d'utiliser une écriture compatible avec le XML, Notamment:

    • Noms de balises et d'attributs en minuscules (le XML est sensibles à la casse). (attention toutefois…)
    • Les attributs, même booléens, ont toujours une valeur, vide si besoin.
    • Les commentaires ne contiennent pas la séquence --.

    Note: Plutôt que la syntaxe XHTML, il est souvent plus commode d'intégrer les données dans un format directement utile (JSON-LD, RDFa…), d'autant que les règles précédentes sont souvent suffisantes pour assurer l'interprétation du document. La syntaxe XHTML N'EST PAS RECOMMANDÉ pour les concepteurs débutants.
